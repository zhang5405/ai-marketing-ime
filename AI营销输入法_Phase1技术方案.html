1. 许可证分析（必须先搞清楚）

关键发现：Trime 和 librime 许可证不同，直接影响商业化

调研后发现一个致命差异：

组件 许可证 商用闭源 说明
Trime（同文输入法 App） GPL-3.0 不行 Fork 后修改并发布，必须开源全部代码，包括你的 AI 话术逻辑
librime（RIME 引擎核心库） BSD-3-Clause 可以 只需保留版权声明，可闭源商用，无传染性
librime-android（Android 预编译包） BSD-3-Clause 可以 社区维护的 Android AAR 打包，许可证跟随 librime
对你的影响

Phase 1（自用）：GPL-3.0 不影响你——因为 GPL 义务仅在「分发」时触发，自己装自己手机上用，不分发 = 无义务。

Phase 2/3（商业化）：如果继续用 Fork Trime 的方式，必须公开全部源码（包括 AI 话术、后端对接逻辑）。这在商业化阶段是不可接受的。

推荐策略：分两步走

Phase 1：直接 Fork Trime，快速验证产品想法。自用不分发，无 GPL 风险。

Phase 2 前：迁移到 librime-android（BSD）+ 自研 IME UI 壳。librime 负责拼音/笔画引擎，UI 完全自己写，彻底脱离 GPL 约束。这一步预计额外 3-4 周，但一劳永逸。

这样你用 Trime 省下的 3-5 个月引擎开发时间，在商业化前花 3-4 周迁移就能洗白，ROI 很高。

3. 本地服务器部署方案

核心思路：后端跑在你自己的电脑上，手机通过 Tailscale 访问，零服务器费用

你的电脑（Windows 10, 8GB RAM）足够运行 FastAPI 后端（仅需 300MB 内存）。AI 推理交给 DeepSeek 云 API，不需要本地跑大模型。

3.1 部署架构

组件运行位置资源配置说明
FastAPI 后端 你的电脑（Windows） 300MB 内存 话术 API + AI 代理，python main.py 直接运行
PostgreSQL 数据库 你的电脑（Windows） 200MB 内存 话术库存储，本地安装即可
话术管理后台 你的电脑（浏览器） — localhost:8000/admin 直接访问
DeepSeek API 云端（DeepSeek 服务器） — 你的后端转发请求，密钥存在本地 .env
Tailscale 手机 + 电脑各装一个 几乎零开销 免费 VPN，自动穿透内网，手机随时随地连

3.2 Tailscale 网络配置

为什么选 Tailscale：

  • 免费（个人版 100 台设备）
  • 零配置：装上登录同一账号，自动组网
  • 自动 NAT 穿透：不管你在哪个 WiFi/4G/5G，都能直连
  • 比 frp/ngrok 简单 10 倍，比公网服务器安全（不暴露端口）

安装步骤：

  1. 电脑：访问 tailscale.com/download，下载 Windows 版安装
  2. 手机：应用商店搜索「Tailscale」安装
  3. 两端用同一个账号登录（Google/微软/GitHub 任选）
  4. 电脑打开终端运行 tailscale ip，记下 IP（如 100.64.0.1）
  5. 手机浏览器访问 http://100.64.0.1:8000 验证连通

App 端配置：server_url = "http://100.64.0.1:8000"（替换为你电脑的实际 Tailscale IP）

6. 键盘布局方案

Trime 键盘布局定制原理

Trime 的键盘完全由 YAML 配置文件驱动，不需要改 Java/Kotlin 源码：

  1. 键盘上每个按键在 .trime.yaml 中定义
  2. 按键可以绑定 command（发送字符/执行动作/切换键盘）
  3. 支持多键盘切换（主键盘 → 话术键盘 → 返回主键盘）
  4. 支持自定义候选栏组件

6.1 键盘布局设计

在标准拼音键盘基础上，修改底部工具栏，加入两个按钮：

# custom.trime.yaml — 自定义键盘布局
keyboards: [ default, huashu, ai_polish ]

default:
  name: "主键盘"
  keys:
    # ... 标准拼音键位 ...
    # 底部工具栏加入两个按钮
    - { click: "commit_text", label: "话术", command: "set_keyboard=huashu" }
    - { click: "commit_text", label: "AI", command: "set_keyboard=ai_polish" }

huashu:
  name: "话术面板"
  rows:
    # 分类 Tab 行
    - [ { label: "全部" }, { label: "促单" }, { label: "售后" }, ... ]
    # 话术列表行（通过自定义 CandidateView 渲染）
    # 底部返回按钮
    - [ { label: "← 返回", command: "set_keyboard=default" } ]

6.2 话术面板交互流程

步骤用户操作系统响应
1在任意 App 输入框中唤起键盘键盘弹出，底部显示「话术」「AI」按钮
2点击「话术」键盘切换到话术面板，显示分类 Tab + 话术列表
3点击分类 Tab（如「促单」）列表筛选到该分类话术
4点击某条话术话术文本直接输入到当前输入框，键盘自动切回主键盘
5点击「← 返回」切回主键盘，不输入任何内容

6.3 AI 润色交互流程

步骤用户操作系统响应
1在输入框中输入一段文字（如 "这个多少钱"）正常打字
2选中输入框中的文字系统高亮选中文本
3点击键盘「AI」按钮键盘弹出 AI 面板，显示 "AI 润色中..."
4等待后端调用 DeepSeek，SSE 流式返回润色结果，逐字显示在 AI 面板
5看到结果满意点击「使用」按钮，替换输入框中的原文本
6不满意点击「重新生成」或「取消」

7.1 后端 PostgreSQL

-- 话术分类表
CREATE TABLE script_category (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(64) NOT NULL,      -- 分类名称：促单、售后、客户跟进
    sort_order  INT DEFAULT 0,              -- 排序权重
    icon        VARCHAR(256),                   -- 分类图标 URL（可选）
    created_at  TIMESTAMP DEFAULT NOW(),
    updated_at  TIMESTAMP DEFAULT NOW()
);

-- 话术表
CREATE TABLE script (
    id            SERIAL PRIMARY KEY,
    category_id   INT REFERENCES script_category(id) ON DELETE CASCADE,
    title         VARCHAR(128) NOT NULL,    -- 话术标题（列表展示用）
    content       TEXT NOT NULL,             -- 话术正文
    tags          VARCHAR(256)[],               -- 标签数组，用于搜索
    sort_order    INT DEFAULT 0,
    is_active     BOOLEAN DEFAULT TRUE,     -- 是否启用
    use_count     INT DEFAULT 0,              -- 使用次数（排序参考）
    created_at    TIMESTAMP DEFAULT NOW(),
    updated_at    TIMESTAMP DEFAULT NOW()
);

-- AI Prompt 模板表
CREATE TABLE ai_prompt_template (
    id            SERIAL PRIMARY KEY,
    name          VARCHAR(64) NOT NULL,      -- 模板名称：润色、扩写、缩写、改语气
    system_prompt TEXT NOT NULL,            -- System Prompt
    model         VARCHAR(32) DEFAULT 'deepseek-chat',  -- deepseek-chat / deepseek-reasoner
    temperature   FLOAT DEFAULT 0.7,
    max_tokens    INT DEFAULT 1024,
    is_active     BOOLEAN DEFAULT TRUE,
    created_at    TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_script_category ON script(category_id);
CREATE INDEX idx_script_tags ON script USING GIN(tags);
CREATE INDEX idx_script_active ON script(is_active, sort_order);

8.4 后端 AI 代理核心逻辑（Python 伪代码）

from fastapi import APIRouter
from fastapi.responses import StreamingResponse
import httpx

router = APIRouter()
DEEPSEEK_API_KEY = "sk-xxx"  # 从环境变量读取，绝不硬编码

@router.post("/api/ai/polish")
async def ai_polish(req: PolishRequest):
    template = await get_template(req.template_id or 1)

    async def stream():
        async with httpx.AsyncClient() as client:
            async with client.stream(
                "POST",
                "https://api.deepseek.com/chat/completions",
                json={
                    "model": template.model,        # "deepseek-chat" (V3)
                    "messages": [
                        {"role": "system", "content": template.system_prompt},
                        {"role": "user", "content": req.text},
                    ],
                    "temperature": template.temperature,
                    "max_tokens": template.max_tokens,
                    "stream": True,
                },
                headers={"Authorization": f"Bearer {DEEPSEEK_API_KEY}"},
                timeout=60,
            ) as resp:
                async for line in resp.aiter_lines():
                    if line.startswith("data: "):
                        # 转发 DeepSeek SSE → 客户端 SSE
                        yield f"data: {line[6:]}\n\n"

    return StreamingResponse(stream(), media_type="text/event-stream")

DeepSeek API 不稳定 中 AI 功能不可用 后端加重试 + 超时处理；可选备用模型（通义千问免费额度）

Tailscale 连接不稳定 低 偶发连接超时 Tailscale 稳定性很高（基于 WireGuard）；备用方案：同一 WiFi 直连电脑局域网 IP
8GB 内存不够用 低 系统卡顿 FastAPI+PostgreSQL 仅占 500MB；不跑本地大模型就没问题

12. Phase 2 迁移路径（脱 GPL + 上云）

Phase 2 需要两件事：① 脱离 GPL（换 librime-android） ② 本地上云（改配置）

12.1 脱 GPL（换引擎）

步骤工作内容预估工时
1引入 librime-android AAR，初始化 RIME 引擎2-3 天
2自研 InputMethodService 壳，接管 onCreateInputView3-5 天
3自研键盘 UI（Jetpack Compose），复用 Phase 1 的话术面板和 AI 面板5-7 天
4RIME 方案配置迁移（拼音词典、候选逻辑）2-3 天

librime-android 项目地址: https://github.com/HNIdesu/librime-android（社区维护，BSD 许可证，提供预编译 AAR）

12.2 本地 → 云服务器迁移

步骤工作内容预估工时
1购买云服务器（2核4G）+ 域名 + ICP 备案1-2 周（备案审核）
2服务器安装 PostgreSQL + Python，部署 FastAPI 代码0.5 天
3配置 Nginx + HTTPS 证书（Let's Encrypt 免费）0.5 天
4App 端修改 BASE_URL：从 Tailscale IP 改为 https://api.yourdomain.com5 分钟
5数据迁移：导出本地 PostgreSQL → 导入云 PostgreSQL1 小时
合计代码零改动实际开发 1 天 + 备案等待
本地部署的迁移优势

因为 Phase 1 本地部署时代码和云部署完全一样，只是配置不同。迁移上云时：

  • 后端代码：零改动，直接 git push 到服务器
  • App 代码：零改动，只改 .env 里的 BASE_URL
  • 数据库：pg_dump + pg_restore 一行命令迁移
  • 不需要重写任何业务逻辑
