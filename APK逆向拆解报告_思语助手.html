11安全风险评估

🔴
高风险：明文 HTTP 流量
usesCleartextTraffic=true 且未配置 networkSecurityConfig，所有网络请求可走 HTTP 明文。虽然主 API 域名使用 HTTPS，但 bebu.vip 的公告接口为 HTTP，登录态/数据可能被中间人窃取。
🔴
高风险：测试框架残留 Release 包
androidx.test.core.app.InstrumentationActivityInvoker 的 3 个 Activity 被导出并注册了 LAUNCHER category，属于测试代码未在 release 构建中移除，可能被用于绕过正常启动流程。
🟡
中风险：大量业务 Activity 导出
18 个业务 Activity 设为 exported=true 但无明确权限保护，可被第三方应用通过 Intent 调起，可能泄露业务页面或被钓鱼利用。
🟡
中风险：allowBackup=true
应用数据可被 adb backup 导出，含用户登录态、企业配置、话术等敏感数据。
🟡
中风险：敏感权限组合
RECORD_AUDIO + SYSTEM_ALERT_WINDOW + REQUEST_INSTALL_PACKAGES + 后台启动库(bgstart.library)，组合起来具备较强的后台行为能力（语音监听 + 悬浮窗 + 静默安装 + 后台拉起），需关注实际使用场景是否合理。
🟢
正面：核心词典加密
主词典 xiaoguandb.db 使用 SQLCipher 加密，有效保护核心 IP；签名完整(v1+v2+v3)；非 debuggable。

12拆解结论

应用画像

思语助手是一款由杭州一苇一豆开发的营销型 AI 输入法，包名后缀 .toc 表明这是面向消费者的版本（推测另有 B 端管理版本）。它将传统输入法（拼音/笔画/英文/手写/语音）与营销话术库 + AI 自动回复深度结合，主打"销售/客服边聊天边营销"场景。

核心差异化

  • 键盘即营销台：在输入法键盘中集成 AI Agent、话术库、场景推荐，无需切换 App
  • B2B2C 闭环：企业配置文章/话术 → 成员通过微信绑定使用 → 邀请分销变现
  • 多模态输入：5 种输入方式（含腾讯云实时语音 + Google 手写识别）
  • 微信生态深度绑定：登录、支付、分享、扫码全链路接入

工程评价

  • ✅ 技术栈现代（Kotlin + Compose + Coroutines），工程化程度高
  • ✅ 核心 IP（词典）有加密保护，签名完整
  • ✅ 支持 Baseline Profile 启动优化，4 套 ABI 全覆盖
  • ⚠️ 安全配置粗放：明文流量未禁、测试代码残留、Activity 过度导出
  • ⚠️ 权限组合偏激进（录音+悬浮窗+安装+后台启动），作为输入法需审慎
